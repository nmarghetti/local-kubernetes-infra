Here is the list of all services to access localarch

Direct access from docker compose

  • Localarch nginx home - http://localhost:30010/
  • Portainer - http://localhost:30001/
  • Gitea - http://localhost:30002/
  • Docker registry - http://localhost:30004/
  • Helm registry - No access
  • Whoami - http://localhost:30009/
  • Podinfo - http://localhost:30008/
  • Dkd - http://localhost:30011/
  • Dkd swagger - http://localhost:30011/dkd/docs/
  • Traefik dashboard - http://localhost:30013/
  • Dnsmasq - http://localhost:30007/
  • Minikube kubernetes server api - http://localhost:30771/version
  • Minikube dashboard - http://localhost:30083/api/v1/namespaces/kubernetes-dashboard/services/http:kubernetes-dashboard:/proxy/#/workloads?namespace=_all (If you added the parameter --minikube-dashboard)
  • Access to any kubectl port forward made on ports from 30050 to 30059 (eg. from WSL: kubectl port-forward -n default services/kubernetes --address 0.0.0.0 30050:443 )
    • http://localhost:30050/ , http://localhost:30051/ , http://localhost:30052/ , http://localhost:30053/ , http://localhost:30054/
    • http://localhost:30055/ , http://localhost:30056/ , http://localhost:30057/ , http://localhost:30058/ , http://localhost:30059/

Access to minikube kubernetes server api if minikube, nginx and traefik are started

  • Minikube k8s - http://nginx-minikube-k8s.localhost:30000/version - http://localhost:30000/nginx-minikube-k8s/version

Access to localarch minikube cluster directly through traefik from docker compose

You can run the traefik-minikube-vault-helm flux playground with the following command: ./docker_build.sh --scenario traefik-minikube-vault-helm --clean registry --services portainer,gitea,registry,registry-ui,helm,nginx,traefik

  • Traefik dashboard - http://dashboard.traefik.minikube.localhost:30000/dashboard/
  • Whoami - http://whoami.traefik.minikube.localhost:30000/
  • Podinfo - http://podinfo.traefik.minikube.localhost:30000/
  • Nginx server with automatic docker image update from flux - http://flux-automated.traefik.minikube.localhost/
  • Flux webhook - http://flux-webhook.traefik.minikube.localhost:30000/ - that works only with POST method, you can try this to notify flux to reconciliate: curl -sSf -X POST "http://flux-webhook.traefik.minikube.localhost:30000/hook/254ac6f85aab9ac284f9990b5349623be0c3471ff894abc568dc28d3419cd49e" -H 'accept: application/json' -H 'Content-Type: application/json' -d '{"events": []}'
